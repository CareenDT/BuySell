Бриф проекта: «Купи-Продай» (Маркетплейс C2C с эскроу-счетом)
1. Базовая архитектура и авторизация
Цель: Регистрация пользователя с подтверждением по email.

Страница регистрации (/signup):

Поля: Email, Пароль, Имя (или Никнейм).

Кнопка «Зарегистрироваться».

Валидация на фронте: проверка формата Email, сложность пароля (мин. 6 символов).

Механизм подтверждения почты:

После нажатия кнопки пользователь получает на Email 6-значный код (одноразовый токен).

Страница ввода кода (/verify): Поле для 6 цифр + таймер обратного отсчета на 5 минут + кнопка «Отправить повторно».

Важно: Если код не введен, аккаунт не активирован, вход на главную / невозможен.

Вход (/signin): Стандартная форма Email + Пароль.

Восстановление пароля: Сценарий сброса через ссылку на почте.

2. Главное меню (Личный кабинет и Лента)
Цель: Точка входа в мир покупок и продаж.

После входа пользователь попадает на /dashboard. Интерфейс разделен на две части:

Верхнее меню (Навигация):

Логотип.

Кнопка «+ Подать объявление» (заметная, яркого цвета).

Иконка уведомлений.

Иконка профиля (выпадающее меню: Мои объявления, Баланс, Настройки, Выход).

Тело страницы:

Бесконечная лента объявлений (Feed).

Фильтры: Сортировка по: Новые / Дешевле / Дороже.

3. Публикация объявления (Многошаговая форма)
Цель: Понятное создание карточки товара без путаницы.

Этап 1: Медиа и заголовок

Загрузка фото: Drag-and-drop или кнопка выбора файла. Обязательно минимум 1 фото, максимум 10. Первое фото становится обложкой.

Поле «Название»: Строка до 100 символов.

Этап 2: Детали сделки

Описание: textarea с авто-растяжением высоты. До 3000 символов.

Цена (₽): Числовое поле. Не может быть 0.

Категория: Выпадающий список (Обязательно поле! Пример: Электроника, Одежда, Детские товары, Хобби). Это задел на будущее для каталога.

Этап 3: Контакты и публикация

Город/Местоположение: Поле ввода с автокомплитом (например, через API DaData или Geoapify).

Состояние товара: Радио-кнопки (Новое / Б/У).

Кнопка «Опубликовать».

4. Система оплаты и защиты (Эскроу / Safe Deal)
Это самый важный пункт в ТЗ. Расписываем детально.

Цель: Деньги переводятся продавцу только после подтверждения получения товара покупателем. До этого они заморожены.

Сценарий покупки:

Покупатель нажимает «Купить с защитой» на странице объявления.

Система показывает сумму: Цена товара + Комиссия сервиса (например, 5%).

Покупатель вводит адрес доставки (если не доставка, то просто ФИО для встречи).

Редирект на платежный шлюз (интеграция с ЮKassa / CloudPayments).

Деньги списываются с карты покупателя и падают на Транзитный счет платформы. Продавец не получает их сразу.

Страница сделки:
После оплаты создается «Заказ №12345» со статусом: «Ожидает отправки».

Продавец видит: «Вам поступил заказ. Отправьте товар / Договоритесь о встрече. Нажмите Отправил после передачи».

Покупатель видит: «Ожидаем отправку. Ваши деньги в безопасности на счете площадки».

Завершение сделки:

Продавец нажимает кнопку «Товар передан».

У Покупателя запускается таймер на 48 часов и появляется кнопка «Подтвердить получение. Претензий нет».

Если нажата «Подтвердить» -> Автоматический вывод средств на баланс Продавца.

Если нажата «Спор / Отмена» -> Заявка уходит в Админку для ручного разбора модератором.

Баланс Продавца:

В разделе «Кошелек» Продавец видит доступную сумму.

Кнопка «Вывести на карту». Деньги уходят с транзитного счета платформы на карту продавца (прямые выплаты физлицам через эквайринг).

Фаза 2: Каталогизация (как на Wildberries)

Умная карточка товара: Вместо просто текста — выбор Цвета и Размера (SKU).

Управление остатками: При создании объявления поле «Количество в наличии». При покупке остаток уменьшается. Если 0 — объявление скрывается.

Поиск: ElasticSearch по названию и описанию с исправлением опечаток и синонимами (искал «айфон 11» -> нашел «iPhone 11»).

Рекомендации: Блок «Похожие товары» внизу карточки товара.

Фаза 3: Логистика и Рейтинг

Трекинг доставки: Интеграция с API СДЭК / Почты России. Генерация трек-номера прямо из интерфейса продавца.

Система рейтинга и отзывов:

После подтверждения сделки Покупатель обязан оставить отзыв (от 1 до 5 звезд).

Продавец получает рейтинг, влияющий на позицию в выдаче (топовые продавцы — выше).

Фаза 4: Веб-версия с каталогом

Адаптация интерфейса под классические сайты объявлений: левое меню с категориями, большие карточки в 4 колонки, галерея фото в превью.

Итоговый стек технологий (рекомендация для разработки)
Backend: Python (Django/DRF)

Frontend: Vue.js (Nuxt) или React (Next.js) для быстрой работы ленты.

База данных: PostgreSQL (обязательно для работы с деньгами).

Платежи: ЮKassa (ЮMoney) — у них есть готовый функционал удержания средств и сплитования платежей.
